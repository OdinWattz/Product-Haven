# Contributing to Product Haven

Thanks for taking the time to contribute.

## Before you start

Please keep contributions focused, practical, and easy to review.

Good candidates for contribution include:

- bug fixes
- security improvements
- compatibility updates for WordPress, WooCommerce, or PHP
- performance improvements
- documentation improvements

## Local setup

For local development, place the plugin in:

`/wp-content/plugins/product-haven/`

Recommended setup:

- WordPress 6.0+
- WooCommerce 7.0+
- PHP 7.4+
- Elementor 3.0+ only when testing frontend widgets

## Pull requests

Before opening a pull request, please:

1. test the main plugin flows manually
2. confirm there are no new PHP notices, warnings, or fatal errors
3. keep naming consistent with the existing `ph_` / `PH_` prefix conventions
4. avoid unrelated formatting changes
5. update documentation when behaviour changes

Small, focused pull requests are preferred over large mixed changes.

## Bug reports

If you are reporting a bug, please include:

- a short summary
- steps to reproduce
- expected behaviour
- actual behaviour
- WordPress version
- WooCommerce version
- PHP version
- relevant screenshots or error messages

## Support expectations

This project is maintained on a best-effort basis. Contributions are welcome, but review times and follow-up are not guaranteed on a fixed schedule.
