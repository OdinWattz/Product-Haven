# Security Policy

## Supported versions

Security fixes, if provided, are handled on a best-effort basis for the latest public version of Product Haven.

## Reporting a vulnerability

If you discover a security issue, please do **not** publish full exploit details in a public issue right away.

Instead, report it privately with:

- a clear summary of the issue
- reproduction steps
- affected versions
- impact assessment if known
- proof of concept only when necessary to validate the report

If no private reporting channel is available yet, the safest temporary option is to avoid disclosing exploit details publicly until there has been reasonable time to review the issue.

## What to expect

- reports may be reviewed when time allows
- validated issues may be fixed in a future release
- response times are not guaranteed

Please keep initial reports responsible and limited to the minimum detail needed for verification.
